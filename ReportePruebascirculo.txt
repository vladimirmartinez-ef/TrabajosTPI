REPORTE DE PRUEBAS

- Pruebas Feature
    En la clase CalcularTest
        Los metodos testcomprobarArea y testcomprobarPerimetro
        se verifica que al hacer post este return en la pagina raiz y se muestre
        un mensaje de respuesta. Esto se hace designando a una variable $response
        que haga post hacia la ruta especificada para cada metodo sea "/area" o
        "/perimetro" y se le mandan parametros dentro de los componentes inputs que
        les corresponden, luego se le asigna a otra variable $response2  que haga
        obtenga o haga get en la raiz "/" para luego que a traves de assertSee 
        compruebe que realmente en la raiz se muestre el mensaje de respuesta 
        correspondiente a los datos introducidos.
    Nota: Este test corre con el siguiente comando:
          php artisan test



- Pruebas Dusk
    En la clase CalcularBTest
        Los metodos tesExample, testExample2 realizan la funcion de verificar que
        al teclear un valor input y presionar el boton correspondiente este muestre
        el mensaje con la respuesta. Esto se logra a traves de dusk que utiliza un
        navegador para que simule ser un cliente, este a traves de los pasos visit
        -que visita la pagina raiz "/", type - que digita en el input de nombre "x"
        el valor 4 o 5, press - que presiona el boton de nombre Area รณ Perimetro, 
        assertSee - que verifica que se muestre en el cuerpo de la pagina raiz el 
        mensaje correspondiente al valor que se digito en el input.
        
        Nota: Para correr este Test, es necesario tener dusk instalado, de no tenerlo
          instalarlo con los siguientes comandos:
          composer require --dev laravel/dusk
          php artisan dusk:install
            Este ultimo comando trae o descarga por default el driver de la version
            89 de google Chrome en caso de tener otra version de chrome tambien
            ejecutar este otro comando sustituyendo el 88 con su numero de version:
          php artisan dusk:chrome-driver 88
            Una vez hecho dentro del repositorio el archivo .env no esta, dado que
            git lo ignora, debe crear el archivo .env justo donde se encuentra el
            archivo .env.example; y copiar todo lo que hay en este ultimo en el 
            archivo creado .env, una vez hecho recuerde ejecutar el proyecto
            para ello puede utilizar el comando:
          php artisan serve
            una vez ejecutado debe abrir el archivo .env y en la variable 
            APP_URL= , en ella debe colocar la url de su servidor local donde se 
            ejecuta su proyecto, y mientras su proyecto este corriendo puede
            ejecutar por fin el test de dusk con el siguiente comando:
          php artisan dusk
